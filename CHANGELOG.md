# 更新日志

本文件记录每次发布的版本与更新内容，**最新版本放在最上方**。
每完成一个需求，就在最新版本条目下加一行简短说明（一句话即可）。

格式约定（发布脚本 `backend/deploy/release.sh` 依赖它解析）：

- 每个版本一个 `## 版本号` 标题，版本号形如 `0.2.0+2`（`x.y.z` 是展示版本名，
  `+N` 是 Android versionCode，必须递增）。
- 标题下用 `-` 列出本次更新内容，每条一句话。
- 顶部第一个 `## ` 条目即「待发布/最新」版本，发布脚本只读它。
- 顶部版本号必须与 `pubspec.yaml` 的 `version:` 一致（脚本会校验，不一致会拒绝发布）。

## 0.2.0+14

- 修复「回忆」性能问题:看完大图返回后所有缩略图重新加载、很卡。原因是缩略图按原图全分辨率(长边 2400px)解码进内存,几张就撑爆图片缓存把缩略图挤掉;改为按实际显示尺寸解码(缩略图小图、大图按屏幕分辨率),返回不再整屏重载
- 修复「回忆」从详情返回时间线时整屏闪一下、缩略图重新加载:之前返回会把时间线重置成加载中再重建;改为首屏才显示加载态,之后的刷新静默后台更新、保留当前列表就地替换

## 0.2.0+13

- 完善账号与安全:新增登录密码。注册新号码 / 已有账号首次登录都需设置密码;登录页加密码输入框,后端只存 scrypt 不可逆哈希(加盐),绝不保存明文。「设置 · 账号与安全」下新增「修改密码」(原密码 + 新密码两次确认)
- 完善账号与安全:请求认证改用登录签发的会话令牌(Bearer),后端只存令牌哈希、可撤销——退出登录即刻失效;生产环境不再接受可伪造的 X-User-Id 旧通道。旧版本需更新到本版重新登录

## 0.2.0+12

- 新增「订阅管家」💳 插件：记录订阅 / 定期账单(视频会员、云盘、房租、宽带等),按月或按年设金额与首次扣费日;到期前自动提醒,到期当天若开启「自动记账」且家庭装了「记账」,会自动把这笔扣费记入账本(订阅分类)并把下次扣费日顺延一个周期;可暂停、可关自动记账。首页卡片显示下一笔最近的扣费
- 「家历」编辑页新增「存入手机日历」开关：保存带日期的日程后弹出手机自带日历的「添加事件」页（预填标题/时间/重复），提醒时间由用户在系统页里自定义；Android 走系统 intent 免授权，iOS 走 EventKit 编辑页
- 新增 AI 通用模块（接 Kimi）：插件可统一调用大模型获取返回内容，支持思考模型；首个落地场景是「看电影」——新增电影只填片名,保存后后台自动用 AI 补上 100-150 字剧情简介、豆瓣评分,并抓取豆瓣海报存到对象存储;列表展示海报缩略图 + 评分 + 简介,补充中有状态提示、失败可一键重试
- 新增「家历」📅 插件：全家共享的日程 + 待办。一条条目选了日期就是日程（可再指定时间），不选日期就是无日期待办；支持每天 / 每周 / 每月重复（重复项「完成」会自动顺延到下次，不被划掉），可指派给家庭成员（显示真实头像）、按「过期 / 今天 / 明天 / 本周 / 以后」分组的议程视图，并能在到期前 N 天推送提醒。首页卡片显示下一件该做的事
- 「记账」新增「软件/订阅」💳 标签（前端 + 后端白名单同步）
- 「囤货铺」新增每周盘点提醒：每周六 20:00 给装了该插件的家庭推送通知，提醒全家清点囤货、补进采买清单（后端定时任务，点通知直达囤货铺）
